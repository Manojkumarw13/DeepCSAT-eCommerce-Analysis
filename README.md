# DeepCSAT - eCommerce Customer Support Analysis

## 🎯 Project Overview

This repository contains the analysis and machine learning workflows for the **DeepCSAT** project, which focuses on evaluating and analyzing **eCommerce Customer Support Data**. The primary goal is to analyze customer support interactions to extract actionable insights, improve customer satisfaction (CSAT), and analyze patterns in support ticket resolution.

## 📊 Dataset

- **File**: `eCommerce_Customer_support_data.csv`
- **Description**: The core dataset comprises customer support interactions, potentially including ticket details, resolution times, customer feedback, and agent performance metrics.

## 📂 Project Structure

- `eCommerce_Customer_support_data.csv`: The primary dataset utilized for the analysis.
- `DeepCSAT – Ecommerce.pptx`: A presentation detailing the project methodology, deep exploratory data analysis (EDA), and key business findings.
- `Sample_ML_Submission_Template-2.ipynb`: The core Jupyter Notebook that encompasses:
  - Data Cleaning & Wrangling
  - In-depth Exploratory Data Analysis (Univariate, Bivariate, Multivariate)
  - Data Visualization & Storytelling
  - Business Impact Analysis

## 🛠️ Tech Stack & Libraries

- **Language**: Python 3
- **Environment**: Jupyter Notebook / Google Colab
- **Common Libraries**: `pandas`, `numpy`, `matplotlib`, `seaborn`, `scikit-learn`

## 🚀 How to Run

1. Clone this repository to your local machine:
   ```bash
   git clone git@github.com:Manojkumarw13/DeepCSAT-eCommerce-Analysis.git
   ```
2. Navigate to the project directory:
   ```bash
   cd DeepCSAT-eCommerce-Analysis
   ```
3. Ensure you have the required Python libraries installed:
   ```bash
   pip install pandas numpy matplotlib seaborn scikit-learn
   ```
4. Launch Jupyter Notebook and open `Sample_ML_Submission_Template-2.ipynb`:
   ```bash
   jupyter notebook
   ```
5. Run the cells sequentially to reproduce the analysis.

## 📈 Methodology

- **Data Preprocessing**: Handling missing entries, duplicate values, and data type formatting.
- **EDA Strategy**: Executing structured visualizations to answer critical business questions and track insights leading to positive or negative growth.
- **Visualization Rule**: Following the "UBM" Rule (Univariate, Bivariate, Multivariate Analysis).
